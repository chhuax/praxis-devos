Use this command to explore a problem space, active change, or design direction without implementing code.

Expect a topic, change name, or question. If context is thin, investigate first and keep the conversation in discovery mode.

Shared workflow expectations:

- Think in public: clarify assumptions, risks, and alternatives.
- Read relevant project and change artifacts before drawing conclusions.
- Use diagrams, comparisons, and option framing when that helps.
- Do not implement application code from explore mode.
- Offer to capture new decisions back into the current change when they harden.
