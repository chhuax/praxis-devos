Use this command to open or advance a proposal in one pass until the change is ready for implementation planning.

Expect a change name or a short feature description as input. If the request is ambiguous, clarify it before creating artifacts.

Shared workflow expectations:

- Create or reuse the target change under `openspec/changes/<change>/`.
- Build the required proposal artifacts in dependency order until the change is apply-ready.
- Read dependency artifacts before writing the next one.
- Keep progress updates short and focused on artifact completion.
- When proposal artifacts are complete, point the user to `/opsx:apply`.
