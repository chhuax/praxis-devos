# 变更协作指南

## 目的

本文档定义日常开发中与契约、文档、排障资料相关的最小协作规则。

## 基本原则

- `contracts/` 是对外行为的事实源。
- 变更完成前，契约、参考文档和必要 guide 必须同步。
- `AGENTS.md` 只负责导航；详细规则集中在本文件和 `docs/checks.md`、`docs/definition-of-done.md`。

## 何时必须更新契约

出现以下变化时，应先更新对应契约：

- 公共 API 路径、请求、响应、鉴权、错误码变化
- 运行时配置项、默认值、取值范围变化
- 权限点、角色绑定、资源范围变化
- CLI 命令、参数、输出格式、退出码变化

## 何时必须更新 guide

出现以下变化时，应更新 `docs/guides/`：

- 用户接入方式变化
- 上手步骤变化
- breaking change 需要迁移说明
- 常见误用模式发生变化

## 何时必须更新 runbook

出现以下变化时，应更新 `docs/runbooks/`：

- 告警阈值变化
- 观测入口变化
- 诊断命令变化
- 回滚条件或恢复流程变化
- 升级路径变化

## 高风险区域

- 认证与授权
- 数据迁移和破坏性变更
- 外部 API 兼容性
- 后台任务与重试机制
- 生产配置变更

## 交付要求

提交变更前，至少确认：

1. 已更新对应契约
2. 已同步参考文档
3. 必要的 guide 或 migration 已补齐
4. 必要的 runbook 已补齐
5. 已通过 `docs/checks.md` 中定义的校验
