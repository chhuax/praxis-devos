# 术语表

## 目的

统一项目术语，减少开发者和 AI 代理对业务词汇的误解。

## 术语

### 契约

系统对外承诺的结构化定义，通常位于 `contracts/` 下，是事实源。

### 参考文档

基于契约整理出的可读说明，通常位于 `docs/reference/` 下，不应替代契约。

### Guide

面向场景和步骤的操作说明，通常位于 `docs/guides/` 下。

### Runbook

面向故障处理、恢复和回滚的可执行手册，通常位于 `docs/runbooks/` 下。

### ADR

Architecture Decision Record，记录关键设计决策的背景、选项和取舍。

### Change

OpenSpec 变更单元，位于 `openspec/changes/<change>/`，表示当前工作上下文。

### Breaking Change

对已有调用方、集成方或使用流程不兼容的变化，需要迁移说明。

### 外部记忆

存放在仓库中的稳定知识系统，包括契约、索引、运行事实、变更记录和操作手册。

## 维护规则

- 新增业务术语时，应优先补充本文件
- 业务词汇应尽量一词一义
- 如果术语与代码命名不一致，应明确映射关系
