# 文档索引

## 目的

本目录存放面向人阅读的文档，这些文档应来自稳定契约或与稳定契约直接关联。

## 结构

- `index.md`：全局入口索引
- `glossary.md`：术语表
- `architecture/`：系统地图、边界、数据流
- `reference/`：系统行为、数据结构、接口和契约的参考文档
- `guides/`：分步骤的接入、上手和操作说明
- `runbooks/`：排障、恢复和回滚流程
- `adr/`：架构与设计决策
- `changelog/`：面向消费者的变化记录
- `migrations/`：迁移说明与破坏性变更记录
- `operations/`：环境、依赖、可观测性和运行事实
- `catalog.yaml`：文档影响映射规则

## 规则

- `contracts/` 是事实源。
- `reference/` 必须与契约保持一致。
- `guides/` 讲流程，不重复堆砌原始字段说明。
- `runbooks/` 必须包含真实可执行的检查和恢复动作。

## `reference/` 边界

允许：

- 对契约信息进行重排，方便阅读
- 对契约内容做直接相关的解释性说明
- 记录与契约直接相关的兼容性备注

不允许：

- 实现细节
- 教程式步骤
- 运维动作
- 与契约无关的背景知识

## `runbooks/` 最低要求

每份 runbook 至少应包含：

- 触发条件或触发阈值
- 观测入口或日志入口
- 诊断命令
- 回滚条件和回滚步骤
- 恢复后验证
- 升级条件
- 事后补充文档位置

## 长期外部记忆建议

为了让 AI 能稳定取回长期知识，建议至少维护：

- `index.md`：让 agent 快速定位关键入口
- `glossary.md`：统一业务术语
- `architecture/system-map.md`：系统边界与模块关系
- `operations/`：真实运行环境事实
- `changelog/` 和 `migrations/`：历史变化和迁移轨迹

这些内容与 `contracts/` 一起构成项目的长期外部记忆系统。
