# Docs Index

## Purpose

This directory contains human-readable documentation derived from or linked to stable contract sources.

## Structure

- `reference/`: exact behavior, shapes, and interfaces
- `guides/`: step-by-step usage and integration
- `runbooks/`: troubleshooting and recovery
- `adr/`: architecture decisions
- `catalog.yaml`: documentation impact mapping

## Rules

- `contracts/` is the source of truth.
- `reference/` must match contracts.
- `guides/` explain workflows, not raw schemas.
- `runbooks/` must contain real checks and recovery actions.
