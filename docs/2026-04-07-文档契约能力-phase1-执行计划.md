# 文档契约能力 Phase 1 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `praxis-devos` 落地轻量版文档契约能力第一阶段，只实现 `docs/codemaps/` 与 `docs/surfaces.yaml` 的安装、同步、最小校验与托管规则提示。

**Architecture:** 这一阶段不做完整文档治理，也不做多目录契约系统。CLI 仍统一收敛到 [`src/core/praxis-devos.js`](/Users/huaxin/Documents/workspace/praxis-devos/src/core/praxis-devos.js)，模板资产采用“轻量 codemap + surfaces 模板”策略，托管规则只负责引导 AI 先读 codemap 和 surface 入口。

**Tech Stack:** Node.js ESM、内置 `node:test`、文件系统模板同步、Praxis 现有 CLI 分发、managed block 注入。

## 1. 已锁定决策

- 第一阶段只做 `docs/codemaps/` 和 `docs/surfaces.yaml`
- 不做 `off / minimal / full`
- 不做 `docs/reference` / `guides` / `runbooks`
- 不做 OpenSpec docs gate
- `surfaces.yaml` 是主对外面统一入口，不预设用户项目一定是 HTTP API
- 主 agent 负责更新 `docs/surfaces.yaml`
- docs 子 agent 负责刷新 `docs/codemaps/**`
- `docs check` 由主 agent 在收尾时执行

## 2. 本阶段范围

### In Scope

- 为目标项目安装最小 codemap 模板
- 为目标项目安装 `docs/surfaces.yaml`
- `setup` / `sync` 补齐这两类资产
- 新增 `docs init`
- 新增 `docs check`
- 托管规则中增加“先读 codemap / surfaces”的提示
- 增加最小测试覆盖

### Out Of Scope

- 细分 `contracts/http`、`contracts/sdk` 等目录
- 完整参考文档体系
- DoD / checks / migration / runbook
- 复杂 diff 推断
- OpenSpec 各阶段文档影响结构化字段
- 自动派发 docs 子 agent
- `docs prepare` / `docs-handoff.yaml`

## 7. docs 子 agent 后续实现说明

本阶段不实现复杂编排器，但需要为后续协作模式留出口。

后续设计方向：

- 主 agent 先判断是否影响主对外面，并更新 `docs/surfaces.yaml`
- docs 子 agent 再根据当前 change 刷新 `docs/codemaps/**`
- 主 agent 最后执行 `docs check`

候选后续能力：

- `praxis-devos docs prepare --change <name>`
- `openspec/changes/<change>/docs-handoff.yaml`

这些能力只记录为后续项，不在本阶段落地。
