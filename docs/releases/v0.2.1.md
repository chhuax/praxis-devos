# v0.2.1 发布说明

发布日期：待发布

## 版本主题

`v0.2.1` 聚焦于把 `0.2.0` 的多 agent 架构进一步收口成更可执行、更可发布的一版：

- 让显式提案入口真正有共享执行层
- 补基础回归测试
- 收紧 npm 包入口与发布元数据

## 重点变化

### 1. 显式提案入口落为可执行命令

新增：

```bash
praxis-devos change --title "Add two factor auth" --capability auth
praxis-devos proposal --title "Add two factor auth" --capability auth
```

其中：

- `change` 是主命令
- `proposal` 是兼容别名
- 默认按更安全的完整提案脚手架创建 `proposal.md`、`tasks.md`、`spec delta`

### 2. OpenCode 插件增加 proposal scaffold 工具

OpenCode 侧新增：

- `praxis-change`
- `praxis-proposal`

它们都复用共享核心，而不是各自维护独立脚手架逻辑。

### 3. 增加基础回归测试

本版本引入最小回归测试，覆盖：

- CLI help 输出
- `change` / `proposal` scaffold 行为
- `list-stacks` 命令

### 4. 修正 npm 包入口元数据

发布入口改为包根目录的 `opencode-plugin.js`，避免发布包继续依赖隐藏目录作为主入口。

## 升级建议

1. 更新到 `0.2.1`
2. 用 `praxis-devos change ...` 取代手工创建提案脚手架
3. 继续在项目内通过 `.praxis/` 调整规则与 skills
