# v0.3.0

## Highlights

- Introduced `npx praxis-devos setup` as the primary onboarding flow for new projects, new machines, and add-agent repair scenarios.
- Added automatic installation for OpenSpec and Codex SuperPowers when Praxis can complete those steps directly.
- Added `npx praxis-devos use-stack <stack>` so framework initialization and stack application are no longer coupled.
- Hardened Windows support for setup/runtime execution, including `npm.cmd` and `openspec.cmd` handling.
- Added install smoke CI across Linux, macOS, and Windows for Codex, OpenCode, and Claude onboarding paths.

## Notes

- `Claude Code` still requires a manual marketplace/plugin install step because Praxis cannot portably automate that platform-specific workflow.
- `doctor` remains the verification and repair check, but primary onboarding guidance is now centered on `setup`.
