# Praxis DevOS v0.2.2

本次版本聚焦两件事：给项目使用者一个真正有用的状态视图，以及把 `0.2.x` 的基础回归测试补到更接近真实工作流。

## 主要变化

### 1. 新增 `praxis-devos status`

这个命令直接面向“已经在一个项目里工作”的场景，输出内容包括：

- 项目是否已经初始化
- `.praxis/` 和 `openspec/` 是否存在
- `manifest.json` 是否存在
- 当前配置了哪些 agent
- 当前项目里有哪些已安装 skills
- 哪些 adapter 已同步
- 当前有哪些未归档的 OpenSpec changes
- `openspec` / `superpowers` 依赖状态

示例：

```bash
praxis-devos status
praxis-devos status --agents codex,claude
```

### 2. OpenCode 插件补齐 `praxis-status`

OpenCode 现在也能通过薄封装工具读取同一份项目状态摘要，不需要用户手工切回 CLI。

### 3. 补强基础测试

新增回归测试覆盖：

- `doctor --strict` 在缺依赖时失败
- `initProject` 生成 canonical `.praxis/` 资产和 agent 入口文件
- `statusProject` 输出初始化状态、configured agents、active changes 和依赖摘要

## 影响

这是一个向后兼容的小版本：

- 不引入新的 stack 配置文件
- 不要求用户为自定义技术栈维护额外元信息
- 更强调项目运行态可见性，而不是框架内部目录枚举
