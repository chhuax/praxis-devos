# praxis-devos v0.2.6

发布日期：2026-03-25

本次发布是 `0.2.5` 之后的能力收紧版本，重点是把 Praxis 对 SuperPowers 的调度从“文档提示”推进到“事件钩子 + 证据校验”，同时修正文档里默认假设全局安装 CLI 的问题。

## 变更摘要

- AI 入口托管区现在来自独立模板文件，而不是硬编码在 JS 中
- proposal flow 新增轻量 `Proposal Intake` 协议：
  - 先静默提取 `change target`
  - `intended behavior`
  - `scope/risk`
  - `open questions`
  - 只有在 intake 暴露阻塞缺口、边界分歧或多方案取舍时，才升级到 `brainstorming`
- SuperPowers 触发方式从静态 skill 表升级为事件钩子：
  - proposal 进入
  - 已批准 proposal 开始实现
  - 多步骤任务
  - bug / 失败测试
  - 可并行子任务
  - 完成前验证
- 新增 `npx praxis-devos validate-session --file <transcript.md>`，可对 transcript 做证据型校验
- 所有用户可执行的文档示例统一改成 `npx praxis-devos ...`
- `init` 现在会修复不完整的 `.praxis/skills/` 安装，而不是遇到空目录直接跳过

## 为什么要改

`0.2.5` 已经把托管区文案收紧到多 agent 友好的口径，但后续验证里又暴露了三类问题：

- SuperPowers 虽然被列进了 skill 表，但缺少稳定触发它们的框架事件点
- transcript 复盘只能靠人工判断，缺少最小自动化校验手段
- 文档里大量直接写 `praxis-devos ...`，会误导用户以为必须或已经做了全局安装

这次版本的目标，就是把这三处从“约定层”推进到“机制层”。

## 影响

这是一个向后兼容的小版本：

- 不改变 `.praxis/` canonical layout
- 不改变多 agent adapter 结构
- 不要求全局安装 CLI
- 主要提升框架入口编排、会话复盘能力和文档可执行性
