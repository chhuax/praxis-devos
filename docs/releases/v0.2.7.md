# praxis-devos v0.2.7

发布日期：2026-03-25

本次发布的重点是把 Praxis 的安装与初始化路径重新收成“按场景使用的一套命令”，让用户不再需要先理解 `bootstrap`、`init`、`stack` 这些内部拆分关系，先把项目和本机环境用起来。

## 变更摘要

- 新增 `npx praxis-devos setup`，作为新项目接入、新电脑 onboarding、补 agent、修复依赖的统一入口
- 新增 `npx praxis-devos use-stack <name>`，把“框架 ready”和“stack ready”拆成两个显式阶段
- `init` 现在可以在没有 stack 的情况下只初始化框架骨架
- Quick Start 和依赖文档改成按场景给命令，而不是让用户先理解底层拆分
- 删除 `--openspec` 的主路径语义；`bootstrap` 现在默认始终包含 OpenSpec
- 当前版本明确按 `macOS / Linux` 支持发布，Windows 兼容性后续单独补齐

## 为什么要改

之前的接入路径虽然功能上可用，但对用户来说暴露了太多框架内部结构：

- 新用户需要先理解 `bootstrap` 和 `init` 的区别
- `--stack` 和基础框架初始化耦合在一起，容易误解为“不选 stack 就不能接入框架”
- `bootstrap --openspec` 在 OpenSpec 已经是强依赖的前提下，变成了一个没有实际语义差异的坏参数

这次版本的目标，就是先把“怎么用起来”这件事收清楚，再继续补 Windows 兼容性。

## 影响

这是一个向后兼容的小版本：

- 保留 `bootstrap` 和 `init`，但把它们降级为底层 / 高级命令
- 旧的 `init --stack ...` 仍然兼容
- 旧的 `bootstrap --openspec` 不再兼容，会返回迁移提示
