# praxis-devos v0.2.5

发布日期：2026-03-25

本次发布是 `0.2.4` 的 follow-up 修正版本，重点是把 AI 入口托管区的文案继续收紧到适合多 agent 运行的口径。

## 变更摘要

- 托管区不再写死类似“当前入口按 codex 处理”的 runtime-specific 表达
- 依赖门禁改成 agent-agnostic 指引，同时明确列出：
  - `npx praxis-devos bootstrap --agent codex`
  - `npx praxis-devos bootstrap --agent claude`
  - `npx praxis-devos bootstrap --agent opencode`
- canonical source 的表述进一步澄清：
  - `.praxis/` 仍然是项目规范事实来源
  - `.opencode/skills/` 仍可作为 OpenCode supplemental layer
  - 但 `.opencode/skills/` 不属于 canonical project state
- 回归测试新增断言，确保这些语义不会再次退回到 agent-specific 或有歧义的版本

## 为什么要改

`0.2.4` 已经把托管区从“大段规则复制”收敛成了短入口摘要，但 follow-up 评审里暴露出两类问题：

- 某些句子仍然隐含“AI 能可靠识别自己当前运行在哪个 agent 下”的假设
- 某些 canonical source 表述容易让人误读成 `.opencode/skills/` 不再生效

对多 agent 项目来说，这两点都会削弱入口门禁的稳定性，所以这次版本专门把它们收干净。

## 影响

这是一个向后兼容的小版本：

- 不改 CLI 结构
- 不改 canonical 目录布局
- 不改 OpenCode 对 `.opencode/skills/` 的补充层支持
- 主要提升托管区作为 AI 入口提示时的稳定性和可解释性
