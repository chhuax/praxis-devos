# v0.2.0 发布说明

发布日期：2026-03-25

## 版本主题

`v0.2.0` 是 Praxis DevOS 从单一 OpenCode 初始化模型，升级为多 agent 架构的第一版正式发布。

这一版的核心目标不是增加更多技能，而是重新定义项目安装态和运行时边界：

- 用 `.praxis/` 统一存放项目级 Praxis 资产
- 用外部 CLI 接管初始化、同步、迁移
- 同时支持 OpenCode、Codex、Claude Code

## 重点变化

### 1. 引入 canonical 目录 `.praxis/`

项目安装态现在统一保存在 `.praxis/`：

- `.praxis/manifest.json`
- `.praxis/framework-rules.md`
- `.praxis/stack.md`
- `.praxis/rules.md`
- `.praxis/skills/`

这意味着 `.opencode/` 不再是事实来源。

### 2. 初始化从插件内工具转为外部 CLI

推荐初始化方式变为：

```bash
npx praxis-devos bootstrap --openspec
npx praxis-devos init --stack java-spring
```

而不是先进入 OpenCode，再依赖插件内 `praxis-init` 做首次安装。

插件工具仍然保留，但现在只是共享核心的薄封装。

### 3. 正式支持多 agent 适配

本版本的适配方式如下：

- OpenCode：插件直接读取 `.praxis/`，并可生成 `.opencode/` 兼容投影
- Codex：通过根目录 `AGENTS.md` 的受管控区块
- Claude Code：通过根目录 `CLAUDE.md` 的受管控区块

### 4. 增加迁移能力

旧项目可直接执行：

```bash
npx praxis-devos migrate
```

迁移命令会把旧 `.opencode/` 项目资产迁移到 `.praxis/`，然后刷新各 agent 适配入口。

## 对用户的影响

### 新项目

建议直接使用：

```bash
npx praxis-devos init --stack <stack-name>
```

### 旧项目

如果项目过去依赖 `.opencode/skills/`、`.opencode/stack.md`、`.opencode/stack-rules.md`，建议尽快迁移：

```bash
npx praxis-devos migrate
```

### OpenCode 用户

OpenCode 仍然支持，但 `.opencode/` 应视为兼容输出，而不是长期维护的项目资产目录。

## 升级建议

1. 先执行 `npx praxis-devos migrate`
2. 检查 `.praxis/manifest.json` 中的 `selectedStack`
3. 检查 `AGENTS.md` 和 `CLAUDE.md` 的受管控区块是否已生成
4. 如有改动 `.praxis/` 内容，执行 `npx praxis-devos sync`

## 对已有 `AGENTS.md` / `CLAUDE.md` 的处理

如果目标项目已经存在这两个文件，Praxis 不会整文件覆盖。

同步策略是：

- 保留用户原有内容
- 只在 `<!-- PRAXIS_DEVOS_START -->` 与 `<!-- PRAXIS_DEVOS_END -->` 之间写入或刷新托管区

## 已知边界

- `AGENTS.md` 与 `CLAUDE.md` 的用户内容和受管控区块是并存关系，后续仍建议持续优化 merge 策略
- `.opencode/` 当前仍会作为兼容投影生成，短期内不会移除
- 当前版本已覆盖 OpenCode、Codex、Claude Code 三类入口，后续若支持更多 agent，将继续沿用 “`.praxis` + adapter” 模型
