# Praxis DevOS v0.2.3

本次版本是一个稳定性补强版本，目标是把 `0.2.x` 的关键工作流测试补到更接近真实使用场景。

## 主要变化

### 1. 补强真实工作流回归测试

新增测试覆盖：

- `syncProject`
- `migrateProject`
- `bootstrapProject`
- `bootstrapOpenSpec`
- `runOpenSpecCommand`

这些测试覆盖了项目初始化之后最常见的维护和适配路径，而不只是脚手架创建。

### 2. 补充 OpenSpec runtime 解析验证

测试现在会显式验证：

- `praxis-devos openspec ...` 优先使用项目本地运行时
- 不会错误落到 PATH 中的全局实现

### 3. CI smoke 补充 `status`

GitHub Actions 的 CLI smoke 现在除了 `help`、`list-stacks` 之外，也会运行：

```bash
node bin/praxis-devos.js status
```

这样可以更早发现项目状态命令的回归。

### 4. 修正 OpenSpec bootstrap 命令引导

针对真实使用反馈，本次也修正了 `bootstrap --openspec` 的提示文案：

- 默认优先提示 `npx praxis-devos openspec ...`
- 只有在 `praxis-devos` 已经安装到 PATH 时，才建议直接使用裸命令

这样可以避免用户通过 `npx` 临时执行后，再误以为本机已经存在全局 `praxis-devos` 命令。

## 影响

这是一个向后兼容的小版本：

- 不改变 canonical 目录结构
- 不引入新的用户配置
- 主要提升发布信心和回归覆盖率
