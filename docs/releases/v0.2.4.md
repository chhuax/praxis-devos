# praxis-devos v0.2.4

发布日期：2026-03-25

本次发布是一个小型稳定性修复版本，重点收敛多 agent 入口文件里的 Praxis 托管区。

## 变更摘要

- `AGENTS.md` / `CLAUDE.md` 中的 `PRAXIS_DEVOS_START` 托管区不再内联整份框架规则和整份技术栈规则
- 托管区改成短入口摘要，只保留 canonical source、路由信息和硬门控
- 读取触发条件被明确写死：
  - 提案、规范变更、提案校验、归档前先读 `openspec/AGENTS.md`
  - 代码实现、测试、重构、调试前先读 `.praxis/rules.md`
  - 选择或调用项目 skill 前先读 `.praxis/skills/INDEX.md`
- 回归测试新增断言，确保托管区不会再次退化成几百行的技术栈全文复制

## 为什么要改

之前的托管区承担了过多内容，既想做 agent 入口，又把完整技术栈细则整段复制进去。这样会导致：

- `AGENTS.md` / `CLAUDE.md` 过长
- 项目入口文件可读性差
- 团队误以为这些入口文件本身就是完整规范来源

这一版把职责重新收敛为：

- 托管区负责把 AI 引导进 Praxis 治理路径
- 完整规则仍然保留在 `.praxis/` 和 `openspec/` 的原始文件里
