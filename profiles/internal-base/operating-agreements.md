# Default Operating Agreements

- Start implementation from the foundation baseline and stack rules before inventing project-local process.
- Clarify ambiguous intent before making irreversible changes.
- Keep edits focused, reviewable, and easy to revert.
- Prefer public, generic conventions in shared baselines; keep private company process in local overlays or project rules.
- State blockers early when access, missing dependencies, or failing verification prevent safe completion.
- Do not mark work complete without reporting verification and any remaining risk.
