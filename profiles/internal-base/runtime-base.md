# ECC Runtime Base

This profile assumes Praxis can run on top of an ECC-oriented internal AI engineering distro.

Current milestone scope:

- treat ECC as the runtime base concept, not a shipped private implementation
- keep all assets generic and editable
- leave integration seams for internal MCP, internal docs, internal commands, hooks, rules, and skills
- seed a public baseline for branch workflow, verification, and operating agreements

Operational guidance:

- day-to-day implementation starts from the foundation profile and project rules
- technology-specific standards still come from the selected stack
- OpenSpec remains available for proposal/governance workflows when teams want formal change control
- local teams can tighten the seeded profile, but the built-in baseline should already be usable on day one
