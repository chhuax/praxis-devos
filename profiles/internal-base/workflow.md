# Daily Workflow Model

Recommended order of attention for projects using this profile:

1. foundation profile and overlays
2. stack rules and project rules
3. project skills
4. OpenSpec, when the work needs governance artifacts

This keeps OpenSpec available without making it the mandatory front door for every daily task.

During implementation, apply the built-in profile baseline in this order:

1. confirm the branch strategy in `branch-workflow.md`
2. follow delivery expectations in `operating-agreements.md`
3. choose and report verification using `verification.md`
